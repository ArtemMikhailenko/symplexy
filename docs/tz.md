ТЗ на розробку сайту Symplexy (Next.js / React)
Тип роботи: Верстка та реалізація лендінгу за дизайном Figma
 Сторінок: 1 довга сторінка (Landing Page) + окремі блоки, що можуть бути повторно використані в майбутньому для інших сторінок

1. Загальна інформація
Потрібно реалізувати лендінг Symplexy згідно з дизайном (Figma).
 Сайт має бути адаптивним, легким, SEO-friendly і виконаним на Next.js (останній App Router).
Сайт містить такі основні блоки:
Header + навігація


Hero (великий банер + фото дівчини)


Логотипи брендів


Популярні нейромережі (карточки)


Інтеграція нейромереж у бізнес (цифри)


Для кого підійде сервіс


Як це працює (інфографіка)


FAQ


Блог (карточки статей)


Блок "Трохи про нас"


Форма зворотного зв’язку


Footer



2. Технологічні вимоги
Обов’язково:

React 19


Typescript

Server Components там, де можливо

3. SEO-вимоги
Семантичний HTML (header, footer, main, section, article, h1–h3)


Правильні метатеги:


title


description


og:image


og:type


canonical


Коректна структура H-заголовків (один H1 — “Сервіс підбору ШІ-рішень…”).


Адаптивні зображення через Next/Image.


Швидкість завантаження: Lighthouse 85+


Вбудований sitemap.xml та robots.txt


Schema.org (Organization + FAQ) — бажано.



4. Адаптивність
Сайт має виглядати однаково якісно:
desktop


tablet


small tablet


mobile


Адаптивність — не спрощена, а повноцінна, як у дизайні:
 коректні відступи, перестановка блоків, розміри типографіки, зображень.

5. Функціональні вимоги по блоках

5.1 Header
Ліворуч — логотип


Праворуч — меню (плавний scroll до секцій)


Кнопки «Для кого», «Про нас», «Блог», «Контакти»


Sticky під час скролу


На мобільному — бургер-меню (анімоване)



5.2 Hero Section
Велике зображення дівчини (png або webp)


Текст + кнопки «Знайти інструмент» та «Дізнатися більше»


Анімація легкого паралаксу (опціонально, обговорити)


Елементи-бульбашки як декоративні



5.3 Слайдер / Логотипи
Логотипи брендів в один ряд


Автоматично скроляться або статичні (узгодити)



5.4 Популярні нейромережі
Grid-картки (6 шт)


Кожна картка:


Іконка


Назва


Короткий опис


Кнопка “Дізнатись більше”


Має бути компонент, який можна перевикористати в майбутньому для каталогу



5.5 Блок “Інтеграція нейромереж”
3 вертикальні статистичні картки


Анімація появи при скролі (fade-in)



5.6 Блок “Для кого підійде”

Підприємці


Продажники


Фрілансери


Команди


Іконки + текст + hover-efffect

5.7 “Як це працює”
Схема з 4 кроків:
Оберіть тип послуги


Знайдіть найкраще рішення


Отримайте поради


Інтегруйте у свій бізнес


Векторна інфографіка

5.8 FAQ

6. Технічні деталі
Картинки
Веб-оптимізація


webp + fallback


Не більше 250–400 KB для великих банерів


Не більше 100KB для іконок


Code quality
Чисті компоненти


Іменування за BEM або CamelCase


Окремі директорії для:
 /components, /sections, /utils, /styles


Продуктивність
Lighthouse ≥ 85


Без блокуючих скриптів


Lazy loading зображень


Використати Next.js Image
